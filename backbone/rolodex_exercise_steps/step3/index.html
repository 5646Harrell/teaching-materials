Exercise 4 - Rendering a Template in a Collection

In this exercise, we will be using the grunt command line utility to pre-compile templates. We'll also render them within a collection.

Part 1 Instructions

  1. Make sure you have npm and the grunt-cli package installed
  2. Download this file
  3. Using command line, cd to the root of the project
  4. Run $npm install to get the necessary node_modules
  5. Run $grunt or $grunt handlebars to compile the template. Remember to do this every time you make a change to the template.

Part 2 Instructions

  1. Open templates/raw/thumb.hbs
  2. Add Handlebars {{expressions}}.
    • Later, you will use JSON to render this template, so make sure that your attributes match. Examples from the models we just made are {{firstName}} and {{lastName}}. Feel free to stick to the attributes given, or switch it up.
  3. Wrap the whole template in an #each Handlebars block helper

Part 3 Instructions

In the last exercise, we rendered a View of a Model. This time, we will render a View of a Collection.

  1. Open SpecRunner.html in a browser
  2. In src/app.js, write code to make all the tests pass.
  3. To see your work, create an HTML page, create <script> tags to bring in the dependencies, along with your code. Append your view to the DOM any way you like!

Bonus - Loop it different ways!

In the above exercises, we used a for loop within a Handlebars #each block. Try moving the loop to different places, like within the collection itself! Don't forget to recompile your template.

Double Bonus - Grunt.js

Use the grunt command line tool to install the package grunt-contrib-watch. Then, configure Gruntfile.js. Configure it so that you can watch for changes on your templates and compile every time you save. Run $grunt watch, or whatever command you've configured, and make changes to your models and templates.
