Exercises

Exercises to help explain how Data Modeling works!

Modeling Places

In this exercise, you'll model two places for storage in a reviews website like Yelp, Google Local, or Foursquare.

We're not terribly worried about format for this. PAIR UP and try to use JSON - but don't worry so much about getting it exactly right. If you get stuck trying to read something you wrote, here's a JSON Parser that will organize things for you.

  • First, create a new file - call it places.json
  • Start with an array with two or more objects - [{}, {}, {}]
  • We're going to model places from this list, or choose your own favorite places!
  • Ask yourself these questions:
    • What properties do each of these places have? An address? A name?
    • How are they different?
    • What makes them different in the same way? (like type: grocery store, or pub, or coffee shop)
    • What information is useful, vs not useful? (phone numbers are useful, but number of bricks isn't. Unless you are building a national brick database.)
    • What is someone looking for when they look up information about this thing? What am I looking for?
  • Try to store some information unique to each place - maybe a section from a menu at a food place, or credentials for the dentist. Models don't always have to be the same, but they do need a "baseline" of similarity.

Modeling People - Exercise 2

In this exercise you'll figure out how to model a person.

  • Create a file, call it users.json
  • In that file, create a blank array, like this: [ ]
  • In that array, create several empty objects. Example:
[
    {
        property: "value",
        property2: ["array", "of", "strings"],
        attribute: 27,
        thing: [24, 36, "47", "mixed array!"]
        sub_objects: [{name: "example", of: "subobjects"}, {remember: "Objects can", go: "in arrays too."}]
    },
    {
        objects: "Don't all have to have the same properties."
    }
]
  • You'll be creating two users modeled after actual people.
  • We want for them to both have usernames, passwords (you can store them as plain text for this exercise.)
  • Other attributes are up to you.
  • We're going to model Oprah Winfrey and Eleanor Roosevelt
  • Use their wikipedia profiles (linked) to gain some info about them (like birthdate, current age) and other trivia.
  • Bonus: Model yourself, and a friend of yours. Use real email addresses.
